## 2. 반복 발견된 고위험 패턴

### 위험 패턴 1: User-Agent에 경쟁 의미 단어 포함
- `OliveDiscountHub` - "Discount" = 가격 경쟁 사이트 오인
- 올리브영이 가장 민감하게 반응할 부분
- **권장 패턴**: `OliveRankingInfoBot`, `OliveDataAggregator`
- **금지 패턴**: Discount, Price, Shop, Deal, Compare

### 위험 패턴 2: 예시 연락처 사용
- `contact@example.com` = 연락 불가능한 가짜 주소
- 투명성 부족으로 악의적 크롤러로 간주
- **필수**: 실제 도메인 + 실제 연락 가능 이메일

### 위험 패턴 3: robots.txt 단순 검증
- 단순 `startsWith()` 매칭은 Wildcard, User-Agent별 규칙 미지원
- **권장**: `robots-parser` 라이브러리 사용
- 실패 시 `return true` → `return false` 변경 (안전 우선)

## 8. 이미지 크롤링 리스크 분석 (Phase 3)

### 핵심 발견 사항
- **이미지는 커머스의 필수 요소** (사용자 요구사항)
- 5가지 방안 중 **안전한 방법은 2가지만** (방안 5 허가 요청, 자체 촬영)

### 치명적 위험 패턴 (절대 금지)
1. **Referer 헤더 조작**: 기술적 보호조치 무력화 → 형사처벌 (3~5년 징역)
2. **프록시 서버 위조**: 컴퓨터 사용 사기 → 형사처벌 (10년 징역)

### 회색지대 방안
- **URL 참조 (방안 1)**: Hotlinking → 전송권 침해 논란 + 차단 가능성 70%
- **썸네일 캐싱 (방안 4)**: 저작권법 제35조의5 공정이용 주장 가능
  - 조건: 검색서비스, 200x200px 이하, 저화질, 비영리
  - 승소율: 50% (법률 자문 필수)
  - 리스크: 패소 시 손해배상 100~500만원

### 권장 3단계 전략
1. **1단계**: 올리브영 허가 요청 (성공률 10~20%, 리스크 0)
2. **2단계**: 자체 촬영 (비용 50~200만원) 또는 브랜드 직접 계약
3. **3단계**: 썸네일 캐싱 (변호사 승소율 60% 이상 확인 시만)
